How This Build Works

Static hosting with browser-side retrieval

GitHub Pages cannot run the original Python and Streamlit stack. This site keeps the same basic workflow by parsing PDFs in the browser, ranking text chunks locally, and sending the final grounded prompt to OpenAI only when you ask a question.

  • Your OpenAI API key is entered at runtime and kept only in this tab.
  • Uploaded PDFs stay in the browser session and are not written back to the repo.
  • The Pages version uses lexical retrieval instead of FAISS embeddings.
